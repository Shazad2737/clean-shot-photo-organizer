### 4.4.2 Detailed Entity Descriptions

#### IMAGE Entity
Represents a photo file in the source folder.

| Attribute | Type | Description |
|-----------|------|-------------|
| `file_path` (PK) | String | Full path to the image file |
| `filename` | String | Name of the image file |
| `extension` | String | File extension (.jpg, .png, etc.) |
| `size_bytes` | Integer | File size in bytes |
| `created_date` | DateTime | File creation date |
| `modified_date` | DateTime | Last modification date |
| `width` | Integer | Image width in pixels |
| `height` | Integer | Image height in pixels |

#### FOLDER Entity
Represents the source folder being processed.

| Attribute | Type | Description |
|-----------|------|-------------|
| `folder_path` (PK) | String | Full path to the folder |
| `name` | String | Folder name |
| `created_date` | DateTime | When folder was first processed |
| `total_images` | Integer | Count of images in folder |

#### CATEGORY_FOLDER Entity
Represents output folders where images are organized.

| Attribute | Type | Description |
|-----------|------|-------------|
| `category_path` (PK) | String | Full path to category folder |
| `category_type` | Enum | Good_Photos, Blurry_Photos, Duplicate_Photos, Face_Photos |
| `folder_path` (FK) | String | Reference to parent folder |
| `image_count` | Integer | Number of images in category |

#### ANALYSIS_RESULT Entity
Stores the analysis results for each processed image.

| Attribute | Type | Description |
|-----------|------|-------------|
| `result_id` (PK) | String | Unique result identifier |
| `image_path` (FK) | String | Reference to analyzed image |
| `blur_score` | Float | Combined blur detection score |
| `is_blurry` | Boolean | Whether image is classified as blurry |
| `is_duplicate` | Boolean | Whether image is a duplicate |
| `duplicate_of` | String | Path to original image if duplicate |
| `has_faces` | Boolean | Whether faces were detected |
| `face_count` | Integer | Number of faces detected |
| `category` | Enum | Final category assignment |
| `processed_at` | DateTime | When analysis was performed |

#### OPERATION Entity
Logs all file operations for undo functionality.

| Attribute | Type | Description |
|-----------|------|-------------|
